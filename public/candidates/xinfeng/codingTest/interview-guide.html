📋 题面 / Problem Statement

English

Title: Implement Matrix Multiplication

Problem:

Given two matrices A and B of real numbers (double precision), implement a function multiply(A, B) that computes and returns their matrix product C = A × B.

  • A has dimensions m × k (m rows, k columns).
  • B has dimensions k × n.
  • The product C has dimensions m × n, where:
C[i][j] = Σp=0..k-1 ( A[i][p] · B[p][j] )

Input specification

  • A: an m × k matrix of doubles, with m ≥ 0 and k ≥ 0.
  • B: a k × n matrix of doubles, with n ≥ 0.

Output specification

  • C: an m × n matrix of doubles.
  • If any input matrix is empty (zero rows or zero columns), return an empty matrix.

Requirements

  1. The function MUST handle non-square matrices (do not assume m == k == n).
  2. The function MUST verify that A's column count equals B's row count. If they do not match, signal an error (throw std::invalid_argument in C++ / raise ValueError in Python).
  3. Implement the algorithm directly. Do NOT delegate the multiplication to a numerical library (e.g., numpy.matmul, Eigen, BLAS, cuBLAS). Library calls are out of scope for this exercise.
  4. Focus on correctness. Performance optimization is addressed in a subsequent phase.

中文

题目：实现矩阵乘法

问题描述：

给定两个实数矩阵 A 和 B（双精度浮点），实现函数 multiply(A, B)，计算并返回它们的矩阵乘积 C = A × B。

  • A 的维度为 m × k（m 行 k 列）。
  • B 的维度为 k × n。
  • 乘积 C 的维度为 m × n，其中：
C[i][j] = Σp=0..k-1 ( A[i][p] · B[p][j] )

输入规范

  • A：维度为 m × k 的 double 矩阵，m ≥ 0，k ≥ 0。
  • B：维度为 k × n 的 double 矩阵，n ≥ 0。

输出规范

  • C：维度为 m × n 的 double 矩阵。
  • 若任一输入矩阵为空（零行或零列），返回空矩阵。

要求

  1. 函数必须支持非方阵（不可假设 m == k == n）。
  2. 函数必须校验 A 的列数等于 B 的行数。若不匹配，需报告错误（C++ 抛出 std::invalid_argument，Python 抛出 ValueError）。
  3. 直接实现算法。不得调用数值库（如 numpy.matmul、Eigen、BLAS、cuBLAS）完成乘法本身。
  4. 本题仅考察正确性；性能优化在后续阶段讨论。

📖 C++ 参考答案 / C++ Reference Answer C++17

推荐这个候选人写 C++ —— 贴他 Azure TTS 的实际工作环境，与后续性能讨论（cache / SIMD / CUDA）无缝衔接。
Recommended language for this candidate — matches his Azure TTS environment and connects seamlessly to the perf phase (cache / SIMD / CUDA).

#include <vector>
#include <stdexcept>

using Matrix = std::vector<std::vector<double>>;

Matrix multiply(const Matrix& A, const Matrix& B) {
    // ── Step 1: 空输入短路 ──────────────────────────────
    if (A.empty() || A[0].empty() || B.empty()) {
        return {};
    }

    // ── Step 2: 提取维度 ────────────────────────────────
    const size_t m = A.size();         // A 的行数
    const size_t k = A[0].size();      // A 的列数 = B 的行数
    const size_t n = B[0].size();      // B 的列数

    // ── Step 3: 维度校验 ────────────────────────────────
    if (B.size() != k) {
        throw std::invalid_argument(
            "dim mismatch: A is " + std::to_string(m) + "x" + std::to_string(k) +
            ", B is " + std::to_string(B.size()) + "x" + std::to_string(n));
    }

    // ── Step 4: 分配输出矩阵，全部初始化为 0.0 ──────────
    Matrix C(m, std::vector<double>(n, 0.0));

    // ── Step 5: 三重循环计算 ───────────────────────────
    for (size_t i = 0; i < m; ++i) {           // 遍历 C 的每一行
        for (size_t j = 0; j < n; ++j) {       // 遍历 C 的每一列
            double acc = 0.0;                  // 局部累加器
            for (size_t p = 0; p < k; ++p) {   // 点积：A 第 i 行 · B 第 j 列
                acc += A[i][p] * B[p][j];
            }
            C[i][j] = acc;                     // 写回 C[i][j]
        }
    }

    return C;
}
📝 全流程解说 / Step-by-step walkthrough

Step 1 — 空输入短路

三个条件：

  • A.empty() → A 一行都没有（m=0）
  • A[0].empty() → A 有行但列数为 0（k=0）
  • B.empty() → B 一行都没有（k=0）

任何一个成立都直接返回 {}（构造空 Matrix），避免后续访问 A[0].size() 时 segfault。

Any empty input short-circuits to {}. Without this guard, accessing A[0].size() below would crash.

Step 2 — 提取维度

用 size_t（无符号整型）保存维度，匹配 STL 容器 .size() 的返回类型，避免编译期 sign-compare warning。const 修饰强调这三个值不应被修改。

Use size_t to match vector::size() return type — avoids sign-compare warnings. const documents intent.

Step 3 — 维度校验

要求 B.size() == k，即 B 的行数等于 A 的列数。不满足就抛 std::invalid_argument，并把具体维度写进错误信息便于调试。

Validate B.size() == k. Throw std::invalid_argument with concrete dimensions in the message for debuggability.

Step 4 — 输出矩阵分配

Matrix C(m, std::vector<double>(n, 0.0)) 这个构造器调用：

  • 外层 vector 大小为 m
  • 每个元素是一个内层 vector<double>，大小 n，所有元素初始化为 0.0
注意 / Note: 内层 vector 是独立堆分配的，行与行之间内存不连续——这是嵌套 vector 数据结构的固有特性，会在 P3 blocking 讨论中重新出现。
Inner vectors are independently heap-allocated; rows are NOT contiguous in memory. This becomes a perf concern in P3 (blocking).

Step 5 — 三重循环

经典 i-j-p 顺序：

  • 外两层 i, j 锁定要算 C 的哪一格
  • 最内层 p 是点积的求和变量
  • 关键工程细节：用局部变量 acc 累加，最后一次性写入 C[i][j]。比直接 C[i][j] += ... 减少对 C[i][j] 的反复 indirection 访问（虽然现代编译器开 -O2 后通常能识别并优化掉，但显式写出是 senior 习惯）。

Classic i-j-p triple loop. Key engineering detail: a local acc accumulator avoids repeated indirection through C[i][j]. Modern compilers often hoist this with -O2, but writing it explicitly is a senior habit.

🐍 Python 参考答案 / Python Reference Answer Python 3.10+

如果候选人选 Python：逻辑可以快速验证，但后续 P1-P5 性能阶段必须切回 C++ 或显式讨论 "翻译成 C++ 后的行为"，否则 Python 解释器开销会掩盖真正的 cache / SIMD 信号。
If the candidate picks Python: fast for verifying logic, but in P1-P5 you must switch to C++ (or explicitly discuss "behaviour assuming a C++ translation") — otherwise Python interpreter overhead masks the real cache / SIMD signals.

from typing import List

Matrix = List[List[float]]

def multiply(A: Matrix, B: Matrix) -> Matrix:
    # ── Step 1: 空输入短路 ──────────────────────────────
    if not A or not A[0] or not B:
        return []

    # ── Step 2: 提取维度 ────────────────────────────────
    m = len(A)              # A 的行数
    k = len(A[0])           # A 的列数 = B 的行数
    n = len(B[0])           # B 的列数

    # ── Step 3: 维度校验 ────────────────────────────────
    if len(B) != k:
        raise ValueError(
            f"dim mismatch: A is {m}x{k}, B is {len(B)}x{n}")

    # ── Step 4: 分配输出矩阵，全部初始化为 0.0 ──────────
    C = [[0.0] * n for _ in range(m)]

    # ── Step 5: 三重循环计算 ───────────────────────────
    for i in range(m):                          # 遍历 C 的每一行
        for j in range(n):                      # 遍历 C 的每一列
            acc = 0.0                           # 局部累加器
            for p in range(k):                  # 点积
                acc += A[i][p] * B[p][j]
            C[i][j] = acc                       # 写回

    return C
📝 全流程解说 / Step-by-step walkthrough

Step 1 — 空输入短路

Python 的 truthiness：空 list [] 为 False。not A 即 len(A) == 0；not A[0] 即 A 的第一行是空 list（k=0）。判断顺序不能颠倒（先确认 A 非空才能访问 A[0]）。

Python truthiness: empty list is falsy. Order matters — must check not A first before accessing A[0].

Step 2 — 提取维度

直接 len()。Python 没有 unsigned int，统一用普通 int。

Step 3 — 维度校验

用 f-string 内插具体维度。抛 ValueError（Python 的内置数值错误异常）。

Step 4 — 输出矩阵分配

[[0.0] * n for _ in range(m)] 是列表推导式：

  • 外层生成 m 个独立列表
  • 每个内层 [0.0] * n 生成长度 n 的全零 list
⚠️ 新手常见陷阱 / Common pitfall: 写成 [[0.0] * n] * m 是错的——这样 m 个内层 list 是同一个对象的引用，改一个全变。必须用列表推导式让每一行是独立的 list。这本身就是一个面试观察点。
Writing [[0.0] * n] * m creates m aliased references to the SAME inner list. Mutating one mutates all. Always use the list comprehension form. This pitfall is itself an observation point.

Step 5 — 三重循环

与 C++ 完全对应。range(k) 生成 0 到 k-1。Python 没有 ++，自然用 for p in range(k)。

🔄 C++ vs Python 两版差异 / Key Differences

维度 / Dimension C++ Python 实际影响 / Practical impact
类型系统
Type system		 强类型，编译期检查；vector<vector<double>> 必须明确 动态类型；Matrix = List[List[float]] 只是 hint，运行时不强制 C++ 类型错编译失败；Python 跑到那行才 TypeError
参数传递语义
Param passing		 必须显式 const Matrix& 才是按引用；写成 Matrix A 就是值拷贝（拷贝整个 m×k 矩阵，O(mk)） list 永远按引用语义传递（无须语法标记） C++ 写错一个 & 就吃 O(mk) 拷贝代价；Python 无此坑
矩阵分配
Matrix allocation		 Matrix C(m, std::vector<double>(n, 0.0))——嵌套构造器，啰嗦但显式 [[0.0] * n for _ in range(m)]——列表推导式，一行 Python 写法更简洁，但有 [[0.0]*n]*m 陷阱
错误处理
Error handling		 throw std::invalid_argument(...)；错误信息要手动拼字符串（std::to_string） raise ValueError(f"...")；f-string 内插 Python 错误信息可读性天然更好
空检查
Empty check		 A.empty() / A[0].empty()——调成员函数 not A / not A[0]——truthiness 隐式判断 Python 写法更短，但隐式判断在大型代码库里降低可读性
索引类型
Index type		 size_t（无符号 64 位），与 .size() 返回值匹配 普通 int，无类型负担 C++ 用 int 会触发 -Wsign-compare 警告
执行性能 (N=1024)
Runtime perf		 编译后 -O2，~1 GFLOPS 量级，几秒内出结果 解释执行 + 每次 += 都涉及 PyObject 装箱，慢 50-100×，可能跑分钟级 写 Python 验证正确性可以，做 perf 题必须切回 C++
内存模型
Memory model		 vector<vector<double>> 每行独立堆分配，行间不连续 list of list of float，每个 float 是 PyObject (24+ 字节)，行间不连续 + 元素本身也是 pointer Python 实际内存占用是 C++ 的 ~5-10×
数值类型
Numeric type		 double 严格 IEEE 754 binary64 float 实际就是 IEEE 754 binary64（与 C double 等价） 数值结果一致，无精度差
编译/运行
Build/run		 需要 g++ -O2 -o mm mm.cpp && ./mm 直接 python mm.py Python 演示更快，C++ 调试反馈链更长
依赖
Dependencies		 仅 <vector> + <stdexcept>，标准库 仅 typing.List（可省略） 两者都零外部依赖
