Task Scheduling with Deadlines

Single-machine scheduling — maximize the number of tasks finished before each task's deadline.

⏱ ~15 min Difficulty · Medium / Hard Greedy · Priority Queue

📋 Problem Statement

You are given a list of n tasks. Each task i is described by:

  • duration[i] — the number of continuous time units required to finish it
  • deadline[i] — the task must be finished at or before this time

Rules

  • You start at time t = 0.
  • You can only work on one task at a time (single machine).
  • Once a task starts, it must run to completion — no preemption, no interleaving.
  • You may pick any subset of tasks and execute them in any order.
  • A task counts as completed only if its finish time ≤ its deadline.
  • You are allowed to skip tasks.

Goal

Return the maximum number of tasks you can complete.

🔧 Function Signature

tasks[i] = [duration_i, deadline_i] · pick your preferred language:

// C++
int scheduleTasks(vector<vector<int>>& tasks);

// C#

# Python
def schedule_tasks(tasks: list[list[int]]) -> int:

📐 Constraints

1 ≤ n ≤ 10^4 1 ≤ duration[i], deadline[i] ≤ 10^4

🧪 Examples

Example 1 
Input:  tasks = [[100, 200], [200, 1300], [1000, 1250], [2000, 3200]]
Output: 3

One valid schedule:
  do   [100,  200]  -> finish at t = 100   (≤ 200  ✓)
  do   [200,  1300] -> finish at t = 300   (≤ 1300 ✓)
  skip [1000, 1250]
  do   [2000, 3200] -> finish at t = 2300  (≤ 3200 ✓)
=> 3 tasks completed.
Example 2 
Input:  tasks = [[1, 2]]
Output: 1
Example 3 
Input:  tasks = [[3, 2], [4, 3]]
Output: 0
(Every task is longer than its own deadline — impossible.)
Example 4 
Input:  tasks = [[5, 5], [4, 6], [2, 6]]
Output: 2
(Best: take [2,6] then [4,6] -> finish at t=2, t=6.
 Taking [5,5] alone gives only 1.)

💬 Discuss Before Coding

  1. In what order should tasks be considered?
  2. When a new task does not fit, can you still improve the schedule?
  3. What data structure helps you answer (2) efficiently?
  4. What is the time / space complexity of your approach?
